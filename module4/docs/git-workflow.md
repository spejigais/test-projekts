# Git darba plūsmas dokumentācija

## Izmantotie soļi:
1. **Strukturēšana**: Izveidotas mapes `module4/src` kodam un `module4/doc` dokumentācijai.
2. **Zara izveide**: Izveidots zars `feature/loops`, lai izolētu jauno funkcionalitāti no `main`.
3. **Failu pārvietošana**: Esošie faili pārvietoti uz `src` mapi.
4. **Iteratīvi commit**: Katra jauna funkcija (fizzbuzz, guess, primes, multiplication) pievienota ar atsevišķu, aprakstošu commit ziņojumu.
5. **Apvienošana (Merge)**: Pēc izstrādes pabeigšanas zars `feature/loops` pievienots `main` zaram.
